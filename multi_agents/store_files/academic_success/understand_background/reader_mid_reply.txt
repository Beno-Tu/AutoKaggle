### Comprehensive Analysis of the "Classification with an Academic Success Dataset" Competition

#### 1. Competition Overview
The competition is part of the Kaggle Playground Series and aims to provide an engaging dataset for practicing machine learning skills. The specific goal is to predict the academic risk of students in higher education, using a classification approach with categories of dropout, enrolled, and graduate. The evaluation metric for submissions is accuracy.

#### 2. Files
- **train.csv**: Contains the training data where the `Target` variable (academic risk) is provided. This dataset is used to train the models.
- **test.csv**: Contains the test data where the `Target` variable is not provided. Participants must predict the target for this dataset.
- **sample_submission.csv**: A template submission file showing the required format for the final submission.

#### 3. Problem Definition
The problem is to classify students based on their academic risk using various features present in the dataset. The model will be evaluated based on its accuracy in predicting the `Target` variable for the test set.

#### 4. Data Information
- **Data Structure**: The datasets are in CSV format and contain multiple features that describe various aspects of students' demographics, academic history, and socio-economic factors.

##### 4.1 Data Types
- **ID Type**: 
  - `id`: Unique identifier for each student (not used in model training).
  
- **Numerical Type**: 
  - `Previous qualification (grade)`, `Admission grade`, `Curricular units (credited/enrolled/evaluations/approved/grade)`, `Unemployment rate`, `Inflation rate`, `GDP`.

- **Categorical Type**: 
  - `Marital status`, `Application mode`, `Course`, `Previous qualification`, `Nacionality`, `Mother's qualification`, `Father's qualification`, `Mother's occupation`, `Father's occupation`, `Displaced`, `Educational special needs`, `Debtor`, `Tuition fees up to date`, `Gender`, `Scholarship holder`, `International`, `Target`.

- **Datetime Type**: 
  - No datetime features are present in the dataset.

##### 4.2 Detailed Data Description
The features encompass demographic and academic background information:
- **Marital Status**: Categorical representation of marital status.
- **Application mode**: Categorical representation of how the student applied.
- **Application order**: Integer indicating the order of application.
- **Course**: Categorical representation of the course applied for.
- **Previous qualifications**: Categorical and continuous features indicating the education level and grades.
- **Family qualifications**: Categorical features indicating the educational attainment of parents.
- **Curricular units**: Various metrics related to students' performance in their courses.
- **Socioeconomic indicators**: Continuous features such as unemployment and inflation rates.

#### 5. Target Variable
- **Target**: Categorical variable that indicates the academic risk classification with three classes: dropout, enrolled, and graduate. This variable is present in the training set but absent in the test set.
