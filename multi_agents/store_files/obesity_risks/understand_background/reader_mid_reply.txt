### 1. Competition Overview
The competition, part of the Kaggle Playground Series, focuses on multi-class prediction of obesity risk factors related to cardiovascular disease. Participants are tasked with utilizing various factors to predict obesity risk in individuals. The competition encourages learning and practice in machine learning using a synthetic dataset, which has been generated to closely resemble real-world data while ensuring the test labels remain confidential. 

### 2. Files
The competition provides three main files:
- **train.csv**: Contains the training dataset with features and the categorical target variable `NObeyesdad`.
- **test.csv**: Comprises the test dataset with the same features as the training data but without the target variable, where participants will make predictions.
- **sample_submission.csv**: Demonstrates the required submission format, showing how to organize predictions for the test dataset.

### 3. Problem Definition
The problem requires participants to predict the obesity risk classification for individuals based on demographic and lifestyle features. The classifications are represented by the target variable `NObeyesdad`, which includes multiple categories of weight status.

### 4. Data Information
#### 4.1 Data Type
- **ID Type**:
  - `id`: A unique identifier for each record (not used in model training).
  
- **Numerical Type**:
  - `Age`, `Height`, `Weight`, `CH2O` (daily water consumption), `SCC` (self-calculated cholesterol), `FAF` (physical activity frequency), `TUE` (time spent in physical activity), `CALC` (caloric intake).
  
- **Categorical Type**:
  - `Gender`, `family_history_with_overweight`, `FAVC` (frequent high caloric food consumption), `FCVC` (frequency of vegetable consumption), `NCP` (number of main meals), `CAEC` (consumption of food between meals), `SMOKE`, `MTRANS` (mode of transportation), and `NObeyesdad` (target variable).
  
- **Datetime Type**:
  - None.

#### 4.2 Detailed Data Description
The dataset includes features that reflect the demographic and lifestyle habits of individuals. Key features include:
- **Gender**: Male or Female.
- **Age**: Age in years.
- **Height**: Height in meters.
- **Weight**: Weight in kilograms.
- **Family History**: Indicates family overweight history.
- **Dietary and Lifestyle Habits**: Frequency of consuming high-caloric food, vegetable intake, meal frequency, and smoking status.
- **Physical Activity**: Frequency and duration of physical activity.
- **Hydration**: Daily water consumption.
- **Cholesterol Level**: Self-reported cholesterol.
- **Caloric Intake**: Daily caloric intake.

### 5. Target Variable
- **NObeyesdad**: This variable categorizes individuals into different weight status classes:
  - Normal_Weight
  - Overweight_Level_I
  - Overweight_Level_II
  - Obesity_Type_I
  - Obesity_Type_II
  - Obesity_Type_III
  - Insufficient_Weight

### 6. Evaluation Metrics
Submissions will be evaluated based on the **accuracy score**, which measures the proportion of correct predictions made by the model.

### 7. Submission Format
The submission file must contain the following columns:
- `id`: Identifier for each record from the test set.
- `NObeyesdad`: The predicted weight status category for each record.
